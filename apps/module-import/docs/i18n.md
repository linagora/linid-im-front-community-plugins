# Module Import — Translation Keys Documentation

This document defines the **complete i18n structure** required by the `module-import` module.

It covers:

- `ImportButton`
- `ImportPage`
- Nested components used inside `ImportPage`
- Status and notification messages

> **📌 Scope:** All keys are scoped under the module `instanceId`.

---

# 1️⃣ Global Structure

All translations must follow this structure:

```json
{
  "[INSTANCE_ID]": {
    "ImportButton": { ... },
    "ImportPage": { ... }
  }
}
```

Where:

- `[INSTANCE_ID]` = module instance identifier defined in route meta
- Example: `moduleUsers`, `users`, `importUsers`

## 3.2 LoadFilesField

Used by `LoadFilesCard` component.

| Key                                                   | Description                       |
| ----------------------------------------------------- | --------------------------------- |
| `[INSTANCE_ID].ImportPage.LoadFilesField.label`       | File input label                  |
| `[INSTANCE_ID].ImportPage.LoadFilesField.loadEmpty`   | Message when no rows detected     |
| `[INSTANCE_ID].ImportPage.LoadFilesField.loadSuccess` | Success message after parsing     |
| `[INSTANCE_ID].ImportPage.LoadFilesField.loadError`   | Error message during file parsing |

| `[INSTANCE_ID].ImportPage.ButtonsCard.clear`          | Clear dropdown label               |
| `[INSTANCE_ID].ImportPage.ButtonsCard.clearAll`       | Clear all rows option              |
| `[INSTANCE_ID].ImportPage.ButtonsCard.clearError`     | Clear error rows option            |
| `[INSTANCE_ID].ImportPage.ButtonsCard.clearImported`  | Clear imported rows option         |

## 3.5 Import Result Notifications

Displayed after batch import execution.

| Key                                      | Description                    |
| ---------------------------------------- | ------------------------------ |
| `[INSTANCE_ID].ImportPage.importSuccess` | All rows imported successfully |
| `[INSTANCE_ID].ImportPage.importWarning` | Partial success                |
| `[INSTANCE_ID].ImportPage.importError`   | All rows failed                |

## 3.6 Clear Notification

| Key                                     | Description                                    |
| --------------------------------------- | ---------------------------------------------- |
| `[INSTANCE_ID].ImportPage.clearSuccess` | Displayed after clearing rows with deletion    |
| `[INSTANCE_ID].ImportPage.clearWarning` | Displayed after clearing rows without deletion |

# 4️⃣ Complete Example (fr-FR)

```ts
export default {
  moduleUsers: {
    ImportButton: {
      label: 'Import users',
    },

    ImportPage: {
      title: 'Data Import',

      LoadFilesField: {
        label: 'Files to import',
        loadEmpty: 'No rows to import',
        loadSuccess: 'File(s) successfully loaded',
        loadError: 'Error while importing files',
      },

      ButtonsCard: {
        cancel: 'Cancel',
        confirm: 'Import',
        confirmLoading: 'Importing...',
        clear: 'Clear...',
        clearAll: 'All rows',
        clearError: 'Error rows',
        clearImported: 'Imported rows',
      },

      ImportedDataTable: {
        expandButtonOpen: 'View errors',
        expandButtonClose: 'Close',
        deleteButton: 'Delete',

        headers: {
          __error: '',
          __delete: 'Actions',
          __status: 'Status',
          __file: 'Source',
          name: 'Name',
        },

        status: {
          READY: 'Ready to import',
          IMPORTING: 'Importing',
          IMPORTED: 'Successfully imported',
          ERROR: 'Import error(s)',
        },
      },

      importSuccess: 'All data has been successfully imported',
      importWarning: 'Some rows could not be imported',
      importError: 'No data could be imported',
      clearSuccess: 'Rows have been removed',
      clearWarning: 'No rows have been removed',
    },
  },
};
```

# 5️⃣ Naming Conventions

- All keys are **scoped under `[INSTANCE_ID]`**
- Component name = translation namespace
- Nested component = nested object
- Status values must match internal enum values exactly:
  - `READY`
  - `IMPORTING`
  - `IMPORTED`
  - `ERROR`

---

# 6️⃣ Summary

The module-import i18n structure is:

```
[INSTANCE_ID]
 ├── ImportButton
 └── ImportPage
      ├── LoadFilesField
      ├── ButtonsCard
      ├── ImportedDataTable
      │    ├── headers
      │    └── status
      ├── importSuccess
      ├── importWarning
      ├── importError
      ├── clearSuccess
      └── clearWarning
```

This structure ensures:

- Scoped modular translations
- Clear separation of component responsibilities
- Full coverage of user-facing text
- Predictable and maintainable i18n architecture
