# Catalog UI - Translation Keys Documentation

This document lists the translation keys used by the catalog-ui shared components. Scopes are provided by the host application; ensure the keys below exist in your locale files.

> **📋 Status:** Keys reflect the current implementation of the catalog UI components and layout.

---

## BaseLayout

**File:** apps/catalog-ui/src/layouts/BaseLayout.vue

| Key                   | Description            | Usage                | Parameters |
| --------------------- | ---------------------- | -------------------- | ---------- |
| `application.title`   | Application title text | Header toolbar title | -          |
| `application.version` | Version badge text     | Header badge label   | -          |

**Example:**

```json
{
  "application": {
    "title": "LinID Identity Manager",
    "version": "v1.0.0"
  }
}
```

| ----------------------------------------------------- | ------------------------------------------ | ------------------------- | ----------------------------------- |
| `[INSTANCE_ID].DropdownButton.title`                  | Dropdown button label                      | Main button label         | -                                   |

## Entity Attribute Fields

**Files:**

- apps/catalog-ui/src/components/field/EntityAttributeTextField.vue
- apps/catalog-ui/src/components/field/EntityAttributeTextAreaField.vue
- apps/catalog-ui/src/components/field/EntityAttributeNumberField.vue
- apps/catalog-ui/src/components/field/EntityAttributeBooleanField.vue
- apps/catalog-ui/src/components/field/EntityAttributeDateField.vue
- apps/catalog-ui/src/components/field/EntityAttributeListField.vue
- apps/catalog-ui/src/components/field/EntityAttributeDynamicListField.vue

> **Note:** `EntityAttributeTextAreaField` uses exactly the same translation keys as `EntityAttributeTextField` (label, hint, prefix, suffix), since it relies on the same i18n schema.

| Key pattern                                                | Description                                                          | Usage                       | Parameters |
| ---------------------------------------------------------- | -------------------------------------------------------------------- | --------------------------- | ---------- |
| `[INSTANCE_ID].fields.[FIELD_NAME].label`                  | Field label (optional)                                               | Input/toggle label          | -          |
| `[INSTANCE_ID].fields.[FIELD_NAME].hint`                   | Helper text (optional)                                               | Input hint                  | -          |
| `[INSTANCE_ID].fields.[FIELD_NAME].prefix`                 | Input prefix (optional)                                              | Input prefix                | -          |
| `[INSTANCE_ID].fields.[FIELD_NAME].suffix`                 | Input suffix (optional)                                              | Input suffix                | -          |
| `[INSTANCE_ID].fields.[FIELD_NAME].close`                  | Close label for the date picker (only for date field)                | Date picker close text      | -          |
| `[INSTANCE_ID].fields.[FIELD_NAME].validation.invalidDate` | Validation message for invalid date format (only for date field)     | Date validation error       | -          |
| `[INSTANCE_ID].fields.[FIELD_NAME].validation.afterDate`   | Validation message when date must be after a reference (date field)  | Date range validation error | -          |
| `[INSTANCE_ID].fields.[FIELD_NAME].validation.beforeDate`  | Validation message when date must be before a reference (date field) | Date range validation error | -          |
| `[INSTANCE_ID].fields.[FIELD_NAME].validation.fromDate`    | Validation message when date must be from a reference onwards (date) | Date range validation error | -          |
| `[INSTANCE_ID].fields.[FIELD_NAME].validation.upToDate`    | Validation message when date must be up to a reference (date field)  | Date range validation error | -          |

> **Note:** Date validation messages can optionally include `{format}` and `{compareTo}` placeholders, which are rendered via Nunjucks at runtime. These are not required and can be omitted if you prefer static messages.

**Example:**

```json
{
  "[INSTANCE_ID]": {
    "fields": {
      "[FIELD_NAME]": {
        "label": "Email Address",
        "hint": "Enter a valid email",
        "prefix": "",
        "suffix": ""
      },
      "[DATE_FIELD_NAME]": {
        "label": "Birth Date",
        "hint": "Enter your date of birth",
        "close": "Close",
        "validation": {
          "invalidDate": "Please enter a date following the format {format}",
          "afterDate": "Date must be after {compareTo}",
          "beforeDate": "Date must be before {compareTo}",
          "fromDate": "Date must be from {compareTo} onwards",
          "upToDate": "Date must be up to {compareTo}"
        }
      }
    }
  }
}
```

## ConfirmationDialog

**File:** apps/catalog-ui/src/components/dialog/ConfirmationDialog.vue

> **Important:** The `i18nScope` passed to this dialog must be dedicated to the dialog instance — not the scope of the calling component. See [ConfirmationDialog documentation](components/dialog/ConfirmationDialog.md) for details.

| Key pattern                                                           | Description                     | Usage                 | Parameters |
| --------------------------------------------------------------------- | ------------------------------- | --------------------- | ---------- |
| `[INSTANCE_ID].[CONFIRMATION_DIALOG_NAME].title`                      | Section title (optional)        | Card header           | -          |
| `[INSTANCE_ID].[CONFIRMATION_DIALOG_NAME].content`                    | Confirmation message (optional) | Text under the title  | -          |
| `[INSTANCE_ID].[CONFIRMATION_DIALOG_NAME].ButtonsCard.cancel`         | Cancel button label             | Cancel button text    | -          |
| `[INSTANCE_ID].[CONFIRMATION_DIALOG_NAME].ButtonsCard.confirm`        | Confirm button label            | Confirm button text   | -          |
| `[INSTANCE_ID].[CONFIRMATION_DIALOG_NAME].ButtonsCard.confirmLoading` | Confirm button loading label    | Confirm button loader | -          |

> `[INSTANCE_ID]` here is the value of the `i18nScope` prop passed when opening the dialog (e.g. `'delete-user-confirmation'`).

**Example:**

```json
{
  "[INSTANCE_ID]": {
    "[CONFIRMATION_DIALOG_NAME]": {
      "title": "Create user",
      "content": "Fill in the form below to create a new user.",
      "ButtonsCard": {
        "confirm": "Save",
        "cancel": "Cancel",
        "confirmLoading": "Saving..."
      }
    }
  }
}
```

## FormDialog

**File:** apps/catalog-ui/src/components/dialog/FormDialog.vue

> **Important:** The `i18nScope` passed to this dialog must be dedicated to the dialog instance — not the scope of the calling component. See [FormDialog documentation](components/dialog/FormDialog.md) for details.

The `FormDialog` component uses the following keys for internationalization:

| Key pattern                                              | Description                           | Usage                     | Parameters |
| -------------------------------------------------------- | ------------------------------------- | ------------------------- | ---------- |
| `[INSTANCE_ID].[DIALOG_NAME].title`                      | Form title (optional)                 | Dialog header             | -          |
| `[INSTANCE_ID].[DIALOG_NAME].content`                    | Form content (optional)               | Text/HTML under the title | -          |
| `[INSTANCE_ID].[DIALOG_NAME].ButtonsCard.confirm`        | Confirm button label                  | Main button               | -          |
| `[INSTANCE_ID].[DIALOG_NAME].ButtonsCard.confirmLoading` | Confirm button loading label          | Main button loader        | -          |
| `[INSTANCE_ID].[DIALOG_NAME].ButtonsCard.cancel`         | Cancel button label                   | Secondary button          | -          |
| `[INSTANCE_ID].[DIALOG_NAME].fields.[FIELD_NAME].label`  | Label for each form field             | Field label (see above)   | -          |
| `[INSTANCE_ID].[DIALOG_NAME].fields.[FIELD_NAME].hint`   | Helper/contextual text for each field | Field hint                | -          |

**Example:**

```json
{
  "[INSTANCE_ID]": {
    "[FORM_DIALOG_NAME]": {
      "title": "Create user",
      "content": "Fill in the form below to create a new user.",
      "ButtonsCard": {
        "confirm": "Save",
        "cancel": "Cancel",
        "confirmLoading": "Saving..."
      },
      "fields": {
        "email": {
          "label": "Email Address",
          "hint": "Enter a valid email"
        },
        "description": {
          "label": "Description",
          "hint": "Enter a description"
        }
      }
    }
  }
}
```
